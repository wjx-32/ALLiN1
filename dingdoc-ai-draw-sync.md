# 一、需求背景
- 在「AI 大模型工具」中，除多模型对话能力外，用户存在**文生图 / 图像生成**诉求，需在同一产品内提供统一的 AI 绘图入口与结果管理。
- 绘图能力与对话能力共享登录与单位额度体系；侧栏「体验版本」区域已约定**每月免费赠送 200 次对话、100 次绘图额度**，次数于每月 1 日重置，当月有效，月底清空（与对话额度规则并列展示，详见侧栏**单一** `?` 悬停说明气泡，多行文案使用 `tip-multi` 样式避免裁切）。
- 本需求文档仅覆盖**前端产品能力**与交互规则，接口字段、计费与后台配置以研发拆分为准。

**（关联）对话首页模型区**：筛选项顺序为**全部、推荐、推理模型、代码开发、国内模型、海外模型**；单会话最多选 **3** 个对比模型，超出未选项置灰；卡片内版本切换与标题区重复勾选等交互以原型为准。

# 二、需求说明/目的
- **目的**：用户可选择多个绘图模型、配置提示词与生成参数，发起生成；在**精简模式**下快速浏览与批量操作，在**详细模式**下按「生成记录」维度查看完整提示词与组图，并支持收藏、筛选、下载与删除等管理操作。
- **范围**：模式切换（AI 对话 / AI 绘图）、绘图左侧控制区、绘图记录侧栏、主区图库（精简/详细）、图片详情弹窗、批量操作与二次确认；不含对话页能力（对话筛选仅上条关联说明）。

# 三、流程说明

## 3.1 入口与布局
1. 用户在左侧 **模式切换** 中点击「**AI 绘图**」。
2. 主区域从「对话首页」切换为「**绘图视图**」；中间出现 **AI 绘图控制面板**（提示词与参数、模型列表、开始生成）；左侧侧栏在「对话记录」与「**绘图记录**」之间切换展示。
3. 用户配置提示词、生成数量、宽高比、分辨率，勾选若干**绘图模型**，点击「**开始生成**」触发任务（原型为模拟 loading 与随机示例图）。
4. 生成结果在右侧主区 **图库** 中展示，支持按记录浏览、搜索、收藏筛选、视图模式切换及单张/批量操作。

## 3.2 与额度的关系（产品规则）
- 绘图消耗**独立剩余次数**（侧栏展示「剩余绘图次数」）；与「剩余对话次数」区分展示。
- 免费额度及重置规则与产品统一说明一致：**每月免费赠送 200 次对话、100 次绘图额度，次数于每月 1 日重置，当月有效，月底清空**（侧栏 `?` 气泡全文）。
- 单次点击「开始生成」具体扣减张数/次数的后端规则需单独立项（例如：按模型数×张数扣减），本需求文档仅要求界面展示**已选模型数**与**预计生成张数**的乘积关系。

# 四、需求风险

| 风险类型 | 风险说明 | 建议 |
|------------|------------|------|
| 成本 | 多模型 × 多图并发会放大 API 成本 | 后端限流、单位/用户额度、失败不重复扣费 |
| 合规 | 生成内容需符合内容安全策略 | 接入审核与敏感词策略（与对话一致或单独策略） |
| 性能 | 高清多图列表易卡顿 | 懒加载、缩略图、分页或按记录折叠加载 |

## 5.2 AI 绘图 — 控制面板

| 页面/区域 | 说明 |
|-------------|------|
| 标题 | 「**AI 绘图**」。 |
| 提示词 | 多行文本框，占位示例：「请输入画面描述，例如：蓝天白云」；默认可预填示例文案。 |
| 生成数量 | **品牌样式自定义下拉**（`draw-dd`），菜单相对触发器**向上展开**，避免遮挡下方表单项。可选 **1～8 连续整数**；**默认 1 张**。标签旁 `?` 使用 `tip-top` 与较高 z-index，保证气泡显示在自定义下拉等控件之上。气泡文案：**每个模型生成图片的数量，可在 1～8 张中选择**。 |
| 宽高比 | 同上自定义下拉，选项：1:1（正方形）、3:4（竖图）、4:3（横图）、9:16（手机竖屏）、16:9（宽屏）。 |
| 分辨率 | 同上自定义下拉；**可选项随当前宽高比变化**（`#drawResMenu` 由脚本根据比例动态生成）。**切换宽高比**时，若当前分辨率不在新比例列表内，则自动落位到该比例下的**推荐默认分辨率**（见下表「默认分辨率」列）。 |
| 统计行 | 展示「已选中模型：N 个」「预计生成：M 张」；**M = N × 生成数量**（与原型一致）。 |
| 开始生成 | 主按钮；点击后触发任务（原型为按钮 loading 约 1.2s 的反馈）。 |
| 模型列表 | 顶部**搜索框**（占位「搜索模型」），下方为可勾选模型行：每行含勾选态、彩色首字母头像、模型名称；支持多选；点击行切换选中。 |

**宽高比 — 可选分辨率 与 切换比例时的推荐默认**（与原型常量 `DRAW_RES_BY_RATIO` / `DRAW_DEFAULT_RES` 一致）

| 宽高比 | 可选分辨率 | 该比例下默认（切换落入） |
|--------|------------|-------------------------|
| 1:1（正方形） | 512×512、768×768、1024×1024 | 1024×1024 |
| 3:4（竖图） | 768×1024、720×1280、1080×1440 | 768×1024 |
| 4:3（横图） | 1024×768、1280×960、1536×1152 | 1024×768 |
| 9:16（手机竖屏） | 720×1280、1080×1920、1440×2560 | 720×1280 |
| 16:9（宽屏） | 1280×720、1920×1080、2560×1440 | 1280×720 |

**绘图模型（原型数据示例，可替换为配置下发）**
- gemini-3.1-flash-image-preview、gemini-3-pro-image-preview、qwen-wanx-v1、doubao-seedream-3、flux-pro-1.1 等。

## 5.3 绘图记录（左侧栏，仅绘图模式）

| 能力 | 说明 |
|------|------|
| 列表 | 展示历史「生成任务」标题列表（如：蓝天白云、极光夜景等），样式同对话侧栏 `nav-item`。 |
| 点击记录 | 主区**详细模式**下滚动定位到对应记录区块；若当前为精简模式，**自动切回详细模式**再滚动。 |
| 高亮 | 支持当前记录高亮（`active`）。 |
| 与收藏筛选 | 若开启「仅展示收藏」导致目标记录下无可见图片，应**自动关闭收藏筛选**或提示，避免无法定位（原型含防御逻辑）。 |

## 5.4 图库主区（右侧）

### 5.4.1 顶栏

| 元素 | 说明 |
|------|------|
| 搜索 | 占位「**搜索生成记录...**」；全链路可为前端过滤或对接搜索接口（待研发定）。 |
| 仅展示收藏 | 开关；开启后仅显示已收藏图片；无数据时展示空态文案。 |
| 精简 / 详细 | 开关切换：默认**详细模式**；精简模式下列表以图片网格为主，详细模式按**记录**分组，每组含标题、日期、长提示词、操作按钮。 |

### 5.4.2 精简模式
- 以**图片卡片**为粒度平铺（多记录合并展示），每张为占位图 + hover 蒙层。
- 交互：
    - **收藏**：点击心形，切换收藏态（已收藏填充红色）。
    - **多选**：悬停/已选时显示左上角勾选框，支持勾选多张，顶部出现**批量操作条**（批量下载、批量删除、清空选择）。
    - **点卡片**：进入**图片详情弹窗**（非点收藏/勾选区域）。
- **批量删除**需**二次确认弹窗**；删除为前端数据移除（接后端时对接真实删除接口）。

### 5.4.3 详细模式
- 按 **记录** 分组，每组包含：
    - 标题、日期、**完整提示词**文案；
    - 操作：**重新生成**（mock 为 loading 后随机换图）、**下载本组**、**删除本组**（需二次确认）；
    - 下方**宫格**展示该记录下所有图片，每张显示模型色点 + 模型名称；若开启仅收藏，组头可显示「已筛选 x / 总 y 张收藏」。
- 单张图同样支持收藏、多选、点击进入详情（同精简模式子集）。

### 5.4.4 图片详情弹窗

| 内容 | 说明 |
|------|------|
| 左侧 | 大图（占位图样式），保持宽高比。 |
| 右侧 | 模型名、宽高比、分辨率、**完整 Prompt**；底部「喜欢/收藏」「下载」；喜欢状态与图库收藏联动。 |
| 关闭 | 右上角关闭、点击蒙层、Esc 关闭。 |

## 5.5 数据与状态（产品约定）
- 每条**生成记录**包含：id、标题、时间文案、**提示词**、图片数组。
- 单张图包含：展示用随机种子/风格变体、**归属模型名**、宽高比、分辨率等，用于列表与详情展示。
- **收藏/多选**为客户端状态；正式环境需持久化到服务端并与账号一致。

## 5.6 非目标（本版不做或原型未实现）
- 控制面板「搜索模型」未接真实过滤（可后续补全）。
- 开始生成、下载、重新生成等均为**模拟**反馈，需对接任务队列与存储后替换。
- 与 OMS 统计、单位管理等后台模块的**绘图次数、金额**对账，复用主项目「统计/提问数据」扩展项另案说明。

# 六、权限说明
- 与「AI 大模型工具」一致：用户需单位开通且账号状态有效方可使用；**绘图**与**对话**可共用产品入口，是否按权限点拆分「仅对话」「仅绘图」由后续运营策略决定，本需求默认**与主工具同开同关**。
